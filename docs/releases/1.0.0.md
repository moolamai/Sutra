# Sutra 1.0.0 release notes

Sutra 1.0.0 is the first frozen release of the Hybrid Cognitive Sync Protocol
and cognitive contract suite. It provides stable cross-language wire shapes,
an executable conformance program, certified local bindings, data-driven domain
packs, and consent-gated learning seams.

The release is cut only by `.github/workflows/release.yml` after the accepted
[Protocol 1.0 freeze RFC](../../rfcs/0001-protocol-1.0-freeze.md) and the
cross-track launch checklist are green. A failed gate blocks publication.

## Highlights

### Frozen protocol and cognitive contracts

- `@moolam/sync-protocol` and `sutra-orchestrator` share
  `PROTOCOL_VERSION = "1.0.0"`. A version mismatch fails closed before merge.
- `SyncRequest`, `SyncResponse`, `CognitiveState`, harness frames, tool
  envelopes, metering events, and degradation modes have committed schemas and
  cross-language fixtures.
- The public memory, model, reasoning, speech, vision, tool, planning,
  knowledge, and runtime interfaces are covered by executable obligations.
- Protocol 1.x evolves additively. Removing, renaming, narrowing, or changing
  the meaning of a frozen field or obligation requires a new major release and
  an accepted RFC.

### One complete cognitive loop

`CognitiveCore` now executes the complete loop:

```text
perceive -> recall -> retrieve -> reason -> plan/act -> respond -> reflect
```

`CognitiveBindings` requires planning and tool bindings in addition to memory,
model, reasoning, and knowledge. Mutating tools are policy-gated and audited
before effects. Overlapping turns are serialized at the plan stage, while
replayed sync remains idempotent.

### Certified native and multimodal bindings

- Model profiles cover desktop llama.cpp, Android ONNX, and Apple silicon MLX.
- Certification combines contract obligations, zero-egress locality proofs,
  and profile-specific performance budgets.
- Indic speech, local vision, and bundled-offline knowledge bindings use the
  same contract and locality model.
- A certification mark is valid only for an exact adapter/profile pair with a
  green report. Reports contain `subjectId`, `deviceId`, hashes, verdicts, and
  measurements—never prompt or utterance bodies.

### Domains are versioned data

- Task graphs use `task-graph.v1`, including pack-owned advance/remediate
  thresholds and a validated acyclic prerequisite graph.
- Knowledge packs use `bindings-knowledge.pack-v1`, with truthful `asOf`,
  locality, source coverage, content shards, and resolvable citations.
- The teacher CBSE slice and doctor formulary sketch are committed flagship
  packs. Domain packages do not become dependencies of the cognitive kernel.

### Sovereign learning seams

- Friction aggregation is subject-scoped and consent-attached.
- Turn trajectories use metadata and content hashes, not prompts, replies,
  keystrokes, utterances, tool arguments, or tool results.
- Capture is admitted through a bounded write-ahead queue and does not wait on
  durable I/O in the turn path.
- Training export produces `training-export.v1` JSONL only for one explicitly
  consented subject at a time. Sutra defines a `FinetuneJob` handoff descriptor;
  it does not upload data or operate a training service.

### Release integrity

The release workflow verifies package readiness, npm/Python version lockstep,
tarball digests, CycloneDX SBOMs, artifact-signing policy, npm provenance, and
rehearsal installation before production upload.

## Breaking changes from 0.x

1. All protocol envelopes and state documents must carry the exact 1.0 wire
   marker. A 0.x document is rejected rather than silently merged.
2. Type compatibility alone no longer qualifies an implementation. Public
   bindings must pass the applicable conformance and locality obligations.
3. `CognitiveBindings.planning` and `CognitiveBindings.tools` are required.
   The core owns plan/act execution and write-ahead tool audit ordering.
4. Hardcoded domain graphs must move to validated `task-graph.v1` data.
   Advance/remediate thresholds belong to the pack.
5. Knowledge corpora must move to `bindings-knowledge.pack-v1`; every returned
   passage needs a citation resolvable through the pack source registry.
6. Learning data cannot be exported merely because it exists. Aggregation,
   trajectory capture, and training export require the correct active,
   subject-matching consent scope.
7. Release versions are lockstep across the TypeScript wire package, Python
   orchestrator, and `PROTOCOL_VERSION`.

See the [0.x to 1.0 migration guide](./MIGRATION-0.x.md) for worked upgrade
steps and validation commands. For operator-facing launch copy, India-first
positioning, and the Certified Binding program entry point, see the
[1.0 announcement pack](./ANNOUNCEMENT.md).

## Compatibility and support policy

- A 1.0 peer may ignore a future optional 1.x field only when that field has a
  documented compatibility path.
- New protocol literals are accepted only after the local schemas are upgraded.
- Deprecated fields remain accepted for at least 180 days or two protocol
  minor releases, whichever is longer; their values are never logged.
- Subject mismatch, consent failure, schema failure, timeout, and locality
  failure remain distinct observable failure classes.

## Tag rehearsal runbook

Release executes only after cross-track-green and A P7 freeze merge. Local
rehearsal before cutting `v1.0.0`:

1. `node scripts/launch-checklist.mjs` — aggregate Track A + B exit gates.
2. `pnpm production-publish:gate` — confirm Protocol 1.0 freeze unlocked.
3. `pnpm release:tag-rehearsal` — P5 publish-pipeline dry-run, verify packages
   and docs land, sign the checklist in
   [`RELEASE-RECORD-1.0.0.json`](./RELEASE-RECORD-1.0.0.json).
4. Tag through `.github/workflows/release.yml` (never a laptop publish).
